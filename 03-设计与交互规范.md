# 03 · 设计与交互规范（几何动态渐变风）

> 本文是 DP 教学网站的**视觉与交互单一事实来源**。风格方向已定：**几何动态渐变风**（大胆几何构成 + 动态渐变网格 + 高饱和配色）。主看设备 = 笔记本电脑，做响应式。技术栈 = React + Vite。

---

## 0. 三条铁律（先立规矩）

本站是**内容 + 交互并重**的教学站，不是艺术站。华丽为学习服务，绝不反噬可读性。

1. **双层分离：舞台层华丽，内容层克制。**
   - **舞台层（Stage）**：首页 hero、各部分封面、区块过渡、页面背景——尽情上动态渐变、几何、粒子、着色器。
   - **内容层（Content）**：正文讲解、代码、公式、题面、表格——高对比、稳定底、克制动效，只保留"进入时轻微揭示 + 交互反馈"。
   - 两层永不混用：渐变绝不铺在正文文字后面（除非叠了足够暗/亮的实底保证对比 ≥ 7:1）。

2. **拒绝"烂大街卡片堆叠"。** 禁止把每节课做成一排排等大圆角卡片。改用：编辑式非对称排版、整屏舞台段、**分栏（讲解 ↔ 实时演示）**、**滚动钉住（sticky scrollytelling）**、有目的的 bento 组合。卡片只在"题目索引"这种真正的列表场景出现，且要有层级差异。

3. **几何即教具。** 几何不是装饰——每个部分的几何母题**直接映射它的 DP 结构**（线性=推进的刻度链、区间=嵌套弧、矩阵=网格、树=分叉、状压=比特点阵）。用户看到封面几何就预感到这类 DP 长什么样。

## 1. 色彩系统

### 1.1 基底与表面（深色沉浸底）
高饱和渐变在深底上最"发光"，且正文用近白字可长时间阅读。默认深色沉浸，另提供浅色阅读模式（正文切浅底深字，舞台层保留）。

| Token | 值（HSL/HEX） | 用途 |
|---|---|---|
| `--canvas` | `#0A0A12` | 全站最底（近黑靛） |
| `--surface-1` | `#12121D` | 大区块底 |
| `--surface-2` | `#1A1A28` | 内容面板 / 玻璃卡底 |
| `--surface-3` | `#242437` | 悬浮 / 代码块 / 输入框 |
| `--border` | `#2E2E44` | 分隔线（低对比） |
| `--text-1` | `#ECEDF5` | 正文主色（对 surface-2 对比 ≈ 13:1） |
| `--text-2` | `#A9ABC4` | 次要说明 |
| `--text-3` | `#6E7091` | 弱化 / 占位 |

> 浅色阅读模式：`--canvas #F6F7FB / --surface-2 #FFFFFF / --text-1 #14141F`，供长文阅读与打印。

### 1.2 七部分渐变身份（几何-渐变导航）
每个 DP 部分一套**专属双色渐变 + 几何母题**，贯穿封面、导航高亮、进度、强调件——形成一眼可辨的路标系统。色相在色轮上尽量拉开。

| 部分 | 主题 | 渐变（起→止） | 几何母题 |
|---|---|---|---|
| **A 线性DP** | 沿链推进 | `#22D3EE → #3B82F6`（青→蓝） | 水平推进的刻度序列 / 一维格带 |
| **B 背包DP** | 装填容量 | `#FBBF24 → #F97316`（琥珀→橙） | 逐格填充的容器 / 方格堆 |
| **C 区间DP** | 嵌套合并 | `#E879F9 → #FB7185`（品红→玫） | 嵌套括号弧 / 区间桥 |
| **D 矩阵DP** | 网格与幂 | `#34D399 → #14B8A6`（翠→青绿） | 方阵网格 / 矩阵块 |
| **E 换根DP** | 旋转视角 | `#6366F1 → #8B5CF6`（靛→紫） | 以不同节点为心的放射树 |
| **F 树形DP** | 自底向上 | `#A3E635 → #22C55E`（黄绿→森绿） | 分叉树冠 |
| **G 状压DP** | 二进制集合 | `#8B5CF6 → #D946EF`（电紫→洋红） | 比特点阵 / 超立方体 |

用法：封面大面积渐变+几何；正文里只作**细节强调**（标题下划渐变、当前节点、进度条、按钮 hover），不铺满。

### 1.3 语义可视化调色板（独立于品牌色，专为演示图例）
所有 DP 演示统一用这套语义色，**跨部分一致**，与品牌渐变分离，保证图例可读。

| 语义 | 色 | 含义 |
|---|---|---|
| 当前状态 current | `#FBBF24` 琥珀（带辉光） | 正在计算的格/节点 |
| 依赖来源 source | `#38BDF8` 亮青 | 转移所依赖的前驱状态 |
| 最优/被选 chosen | `#A3E635` 黄绿 | 被 max/min 选中的转移 |
| 已确定 settled | `#64748B` 板岩灰 | 已算完、不再变 |
| 非法/越界 invalid | `#FB7185` 玫红 | 不可达 / -INF / 越界 |
| 转移边 edge | 白→语义色 流光 | 转移动画的连线 |

## 2. 字体系统

> 具体字体最终以 `04-技术栈与素材清单.md`（技术 agent 核验许可证）为准，此处为选型意图。全部要求 **OFL/SIL 等免费可用（含教育/非商用场景）**。

## 3. 几何语言

- **母题即结构**（见 1.2 表）——封面用大尺度几何预告该 DP 的形态。
- **动态渐变网格（mesh gradient）**：舞台层背景用缓慢流动的网格渐变（着色器或 CSS/SVG，见 §5 性能），叠**细噪点（grain）**去香蕉带（banding），叠**细网格线**呼应"DP 表格"。
- **几何作分隔**：区块之间用斜切 / 弧线 / 折线分隔，代替千篇一律的水平线，但保持克制（1–2 种，不堆砌）。
- **可复用图元**：格子、箭头（转移）、括号弧（区间）、节点-边（树）、比特方块（状压）——做成 SVG/组件库，正文与演示共用同一套视觉语言。

## 4. 动效原则

| 原则 | 落地 |
|---|---|
| 有目的 | 动效只用于：揭示层级、表达 DP 状态变化、给交互反馈。无意义炫技一律砍。 |
| 物理感 | 交互动效走**弹簧（spring）**（Motion）；避免线性生硬缓动。 |
| 滚动叙事 | 讲解用**滚动驱动分步揭示**（GSAP ScrollTrigger）：文字滚动、演示钉住随讲解推进。 |
| 状态动画 | DP 逐格填充/树 DP 冒泡：自建轻量引擎（rAF + Motion values），可暂停/单步/拖拽进度。 |
| 尊重降级 | `prefers-reduced-motion` → 关闭大动画，保留静态终态与单步按钮。 |
| 60fps@笔记本 | 只动 transform/opacity；离屏动画用 IntersectionObserver 暂停；粒子/着色器限量（见 §5）。 |

## 5. 布局系统（反卡片堆叠）

- **舞台段（full-bleed）**：首页 hero、部分封面——整屏，几何渐变主导。
- **分栏教学（split）**：左讲解（可滚动）+ 右演示（sticky 钉住）。笔记本 16:9 天然适配。
- **滚动钉住叙事（scrollytelling）**：一段复杂推导（如背包 2D→1D、换根转移）用钉住的演示 + 分步文字，滚一步演一步。
- **有目的的 bento**：仅在"部分总览""方法论全景"用大小不一、承载不同信息量的分区块，杜绝等大卡矩阵。
- **题目索引**才用规整列表/表格（可筛选：部分/类型/难度/来源年份）。
- **响应式**：桌面（≥1280）双栏；笔记本（1280–1536）为**主要目标**，逐一走查 1366×768 / 1440×900 / 1536×864；平板单栏折叠演示；手机为兜底可读（演示降级为"点击播放"）。

## 6. 核心交互组件 · DP 可视化引擎（全站复用）

一套**参数化引擎**驱动所有部分的演示，避免每题重造。规格：

**能渲染的状态载体**
- 一维数组（线性 DP、背包滚动数组）
- 二维表格（背包 2D、LCS、区间 DP 的 `f[l][r]`、矩阵路径）
- 树（树形 / 换根：节点冒泡、换根前后对比）
- 比特点阵 / 集合格（状压：mask 逐位点亮）

**能演的过程**
- 按 DP 计算顺序**逐格/逐节点填充**；高亮**当前状态**（琥珀）+ **依赖来源**（青）+ **被选转移**（黄绿）。
- 同步显示**正在应用的转移方程**（KaTeX），方程里对应变量随高亮联动。
- **值的来源追溯**：点任意格，回溯它由哪些前驱推来（连线流光）。

**用户可操作（"可改值"硬需求）**
- 编辑输入：改数组/物品重量价值/字符串/树结构/棋盘 → **实时重算并重播**。
- 播放控制：▶ 播放 / ⏸ 暂停 / ⏭ 单步 / ◀▶ 拖拽进度条 scrub / 调速。
- 视图切换：如背包 **2D ↔ 1D 滚动数组** 切换、**顺推 vs 逆推** 并排对照。

**技术载体**：表格/数组/树用 **SVG + Motion**（清晰、可交互、笔记本友好）；大规模粒子/网格背景用 **WebGL（react-three-fiber）**；小游戏视规模选 Canvas2D/SVG。

## 7. 各部分演示与互动游戏清单

每个 DP **类型**至少 1 个"可改值演示"；每个**部分**额外 1 个"精美互动小游戏"（可玩、计分、可"一键展示 DP 最优解"对照玩家直觉）。

| 部分 | 关键演示（可改值） | 部分互动游戏 |
|---|---|---|
| **A 线性** | LIS 耐心排序动画 / LCS 网格回溯路径 / 最大子段和滑动 | **LIS 接龙**：玩家手动挑最长上升子序列，与 DP 最优对拼长度 |
| **B 背包** | 背包表填充 + 2D↔1D 切换 + **顺推/逆推 bug 并排**（呼应"物品重取"陷阱） | **装包大师**：拖物品进背包比价值，实时对照 DP 最优 |
| **C 区间** | 区间 `f[l][r]` 三角表填充 / 括号弧嵌套 / 环形拆链 | **合并石子**：玩家选合并顺序累计代价 vs DP 最小代价 |
| **D 矩阵** | 网格路径热力图 / **矩阵快速幂**：2×2 矩阵平方倍增点亮 | **幂次加速器**：手动选平方/相乘凑幂次，比暴力步数 |
| **E 换根** | **点任意节点换根**，O(1) 换根转移前后箭头 + 距离和实时变 | **换根巡礼**：找使距离和最小的"最优树心"，对照换根答案 |
| **F 树形** | 子树 DP 值后序冒泡 / 没有上司的舞会 邀请开关 / 树上背包容量滑杆 | **舞会邀请**：勾选参会者最大化欢乐值 vs 树形 DP 最优 |
| **G 状压** | 比特点阵逐位点亮 / 棋盘逐行状压合法性判定 / TSP 访问 mask | **棋盘布阵 / TSP 巡回**：放棋子看合法 mask，或连巡回路线对照最短 |

## 8. 页面级组件

- **代码面板**：C++（ShanireZ OJ 风格：Allman 花括号、4 空格、1-based、`endl`、显式 `long long`），Shiki 构建期高亮，含「复制」「在洛谷打开 Pxxxx」按钮、行内注释。
- **题目卡**：题号（渐变徽标）· 题名 · 来源/年份 · 难度 · 考点标签 · 洛谷链接；例题卡展开为「题面摘要 → 状态定义 → 转移方程(KaTeX) → 复杂度 → 代码 → 陷阱」。
- **公式**：KaTeX，与演示步骤联动高亮。
- **导航**：左侧部分导航（A–G，各带自身渐变），顶部进度/面包屑；`⌘K` 全站搜题/跳转。
- **进度追踪**：localStorage 记录已读类型 / 已看演示 / 练习自评勾选。
- **易错点框 / 悟道框**：沿用现有课件"⚠️ 走火入魔 / 💡 悟道"的信息框语义（去修真措辞，改中性："⚠️ 常见陷阱 / 💡 本质"）。

## 9. 可访问性与降级

- 颜色不作唯一信息载体（语义色 + 图标/文字双编码）。
- 对比度：正文 ≥ 7:1，大字 ≥ 4.5:1。
- 键盘可达：演示控制、导航、游戏均可键盘操作。
- `prefers-reduced-motion`：禁大动画/着色器，保留静态与单步。
- 低端 GPU 兜底：检测到掉帧则自动降级 WebGL 背景为静态渐变图。
