# 05 · 内容创作标准（黄金样板：B 背包 01 / 完全）

> 本文是每个 DP **类型页**如何撰写的**单一事实来源**。目的：让 M2 铺开的六大部分与已验收的 B 背包在**结构、深度、模式**上完全一致。
> 标准提炼自 `site/src/content/b/Knapsack01.tsx` 与 `KnapsackComplete.tsx`——owner 已认定这两页「相近程度、深度、模式基本达标」，就此立为标准。
> 关联：`03-设计与交互规范.md`（视觉/引擎）、`02-内容架构与洛谷选题.md`（每题选题）、`handoff/HANDOFF.md §3`（模板契约）、`site/src/components/ui/*`（组件）。

## 0. 一句话

每个 DP **类型** = 一个内容组件，照**同一条章节流水线**写：**场景引入(配图) → 状态转移推导(配图+本质框) → 手算逐步 → 可改值演示 → 深化/对照(再演示) → 例题 → 练习 → 上下页导航**。精讲、图文并茂、逐步推进；例题全洛谷原生题。

## 1. 页面骨架（章节序列 · 照此顺序，勿跳步）

每个 `<section className="lesson">` 以 `<h2 className="section-title">` 领起。下表是**黄金序列**，两个样板逐节对齐：

| # | 章节职责 | 必备元素 | 01 背包对应节 | 完全背包对应节 |
|---|---|---|---|---|
| 1 | **引入 / 动机** | `.prose` 讲一个**具体小例子**（小数据）+ `figure`(SetupFigure) + **为什么朴素解不行**（贪心反例 / `2^n` 爆炸）。**别上来就抛方程**。 | 从「整件取舍」说起 | 同一种物品，取之不尽 |
| 2 | **状态与转移（推导）** | 定状态 `f[…]` 的**中文语义** → `figure`(DecisionFigure) 图示转移分叉 → `<MB>` 转移方程 → 边界 / 答案 → `<InfoBox kind="key" title="本质">` 点破「把指数压成多项式」。 | 状态与转移：取，还是不取 | 与 01 只差一个方向 / 为什么还是 O(nm) |
| 3 | **手算逐步** | `.steps > .step`（编号 `0/1/2/3` 或 `✓/✗`），用**引入时的同一组小数据**把方程「跑起来」；末尾 `.pointer-cue` 指向下方演示。 | 跟着算一遍 | 跟着算一遍：看它把一件反复拿 |
| 4 | **可改值演示** | `.demo > .demo__body > <XxxDemo/>`，DPViz 驱动、输入可编辑、实时重算。 | 看它一格一格长出来 | 看它累积起来 |
| 5 | **深化 / 优化 / 对照** | 该类型的「第二层」：空间优化(2D→1D)、复杂度折叠、易错点、或与近亲**并排对照**（常再来一个演示 + `InfoBox kind="warn"`）。 | 卷成一维 + 为什么不能正推(ForwardBugDemo) | 01 还是完全？并排看(CompleteContrastDemo) |
| 6 | **例题** | `<h2>例题</h2>` + **2–3** `<ExampleCard>`。 | P1048 / P2871 / P1164 | P1616 / P5662 / P5020 |
| 7 | **练习** | `<section className="lesson exercises">` + `<h2>练习</h2>` + **3** `<Exercise>`。 | P1049 / P1417 / P1466 | P2918 / P2725 / P1832 |
| 8 | **章节导航** | `<nav className="type-nav">`，上一 / 下一类型。 | → 完全背包 | ← 01 背包 |

> 序列可**微调**（如第 5 节按类型不同而不同），但 1→4→6→7→8 的主干与「引入必带反例、推导必带配图+本质框、演示必可改值」是硬要求。

## 2. 深度基线（每页的「量」，达不到即偏浅）

- **篇幅**：约 **250–320 行** TSX（含代码常量）。
- **讲解章节**：**6–8** 个 `section.lesson`。
- **配图 `figure`**：**≥3**（引入图 + 转移决策图 + 深化/对照图），皆 **on-brand SVG**（仿 `KnapsackArt.tsx`，用 `var(--accent-1/2)` 随部分变色，**勿用外部图片**）。
- **可改值演示**：**≥1**（理想 **2**：主演示 + 对照/bug 演示）。
- **手算逐步卡**：一组 **3–4 步**，贯穿全页用同一小例子。
- **`InfoBox`**：**≥1** 个「本质」(key)；有坑再 **+1**「常见陷阱」(warn)。
- **例题 2–3 + 练习 3**。
- **交叉链接**：与近亲类型**互链**（01 ↔ 完全 已示范：同一机制在两页「bug ↔ 特性」翻面）。

## 3. 讲解写法（文风 · owner 敏感项）

- **精讲、图文并茂、逐步**——不是纯文字堆砌，不是「简略提要」。每个抽象点尽量配一张图或一段手算。
- **归纳式**：具体小数据入手 → 推出一般方程 → 点破本质。**先讲「为什么需要它 / 朴素为何不行」，再给解**。
- **讲者口吻**带读者一步步推：用「注意…」「盯住 `f[j-w]` 看…」「就这一处方向之差…」等引导语；关键短语 `<strong>` 加重。
- **术语中立**（正常风格，非修真）：用「**常见陷阱 / 本质**」，不用「走火入魔 / 悟道」。
- **例题的「为什么选它」**必须说清该题的**教学独特性**（如「逼你写一维滚动数组」「从最优 DP 跨到计数 DP」「较新 CSP-J 真题代表」「完全背包当可达性判定」）——不是复述题意。
- **练习 `hint`**：一句点破**解法关键**（转化 / 状态设计 / 排序预处理），**不贴完整代码**。

## 4. 硬性技术约束（★ = 已踩坑，勿重蹈）

- **公式**：`import { M, MB } from '../../components/ui/Math'`；`<M>` 行内 / `<MB>` 块级。★**公式内禁中文**（KaTeX 数学字体无 CJK 字形，`\text{中文}` 缺字/方框）——中文标注放 HTML 文本。★**不引 react-katex**（3.1 与 katex 0.17 不兼容，把 `\max\big` 当字面文本渲染）。
- **代码**：`<CodeBlock code={CODE_CONST} luogu="Pxxxx" />`；**ShanireZ OJ 风**（Allman 花括号 / 4 空格 / 1-based / `endl` / 显式 `long long`）——写代码时挂 `shanirez-style` 技能。关键行加**行内中文注释**（如 `// ★逆推：从大容量往小推`）。
- **题目**：例题 / 练习**只用洛谷原生 P/B 题**（见 `02` 逐题核验清单），非 remote judge；`ExampleCard`/`Exercise` 的 `pid` 自动生成洛谷链接。
- **演示**：写步骤生成器 `site/src/components/demos/<topic>/solvers.ts` 产出 `VizModel`（`frames[]`，每帧 `{values,states,arrows,active,formula,caption}`）；再包一个可编辑输入的 Demo，`<DPViz model={…} key={inputsHash}/>`（key 变→remount 重播）。引擎详见 `03 §6` 与 `handoff §3.2`。
- **配色**：组件一律用 `var(--accent-1/2)`、`var(--grad-accent)`——它们由 `[data-part]` 自动切成该部分的**暖矿宝石色**。★强调色上的文字用 `var(--text-on-accent)`（暖黑），**别用白字**。**别在正文文字后面铺渐变**（舞台层华丽 / 内容层克制，见 `03 §0`）。
- ★**不做从 `opacity:0` 起步的进场 / 滚动揭示动画**——无头 / 后台标签页会暂停 CSS 动画与 IntersectionObserver，内容会永久卡在隐藏态（已两次踩坑）。高级感靠配色 / 阴影 / 字体 / hover。
- **正文行宽**：讲解段落放进 `.prose`（行宽受 `--measure` 约束）；★`--measure` 用 **em/rem 而非 ch**（ch 按拉丁 "0" 字宽标定，中文只有约一半字数/行）；例题体内 `.prose` 用 `max-width:none` 占满卡片。

## 6. 注册与状态

1. 内容组件放 `site/src/content/<part>/<Name>.tsx`，在 `site/src/content/registry.tsx` 注册，键 `` `<pid>/<slug>` ``（如 `b/01`）。
2. 建好一个类型，把 `site/src/data/parts.ts` 里它的 `status` 从 `'planned'` 改 `'ready'`（侧栏 / 部分页据此放链接）。
3. 每**部分**额外 1 个互动游戏：`site/src/components/games/<Game>.tsx` + 在 `games/registry.ts` 按 part id 注册（样板 `PackMasterGame.tsx`）。
