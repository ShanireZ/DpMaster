## 三个页面

### 1. `/method` · DP 通用方法论 —— 优先级最高（首页已有入口）
**定位**：全站总纲/入门。新手先读，建立 DP 思维框架。
**内容大纲**：
- 什么是 DP：最优子结构 + 无后效性 + 重叠子问题
- 状态设计：如何定义 `dp[...]`（维度、含义）
- 转移方程：从子问题推当前
- 递推顺序：填表方向（对角 / 逆推 / 记忆化搜索）
- 空间优化：滚动数组 / 一维压缩
- 调试方法：打印 DP 表、对拍、边界检查
- 常见陷阱：初始化、边界、下标越界、正序 vs 逆序
- 七大家族速览 + 跳转（可复用 Home 的 bento 或简版）
**建议复用**：`prose` 容器、`InfoBox`、`Math`（KaTeX）、`CodeBlock`（Shiki）、可选嵌 1–2 个 `DPViz` 演示。撰写套 `05-内容创作标准.md`。

### 2. `/problems` · 题目索引
**定位**：全站洛谷题的汇总检索页。
**数据来源**：`src/data/parts.ts` —— 每个 type 的例题 + 练习（洛谷 P/B 号）。若 parts.ts 未存题号，从 `02-内容架构与洛谷选题.md` 提取，并考虑给 parts.ts 补题号字段。
**内容 / 交互**：
- 按部分（A–G）分组，或按洛谷难度（9 档）筛选
- 每题：题号 + 标题 + 难度 + 所属类型 + 洛谷外链 + 跳到对应类型页
- 可选顶部搜索框
**建议复用**：类型页的题目卡（`components/ui/ProblemBits`）、难度 token。

### 3. `/about` · 关于 · 如何使用
**定位**：站点介绍 + 使用引导 + 致谢/许可。
**内容**：
- 站点是什么、给谁看、覆盖范围（7 部分 37 类型全洛谷原生题）
- 如何使用：交互演示怎么玩（播放 / 单步 / 进度条 scrub / 改值重算）、DP 引擎说明、深浅色切换
- 数据来源与许可：例题取自洛谷原生题库；字体 / 图标 / 音效许可见 `04-技术栈与素材清单.md`
- 反馈入口（`FeedbackWidget` 已全局挂载）

## 接入（每个页面完成后）
1. 在 `src/pages/` 新建组件（如 `MethodPage.tsx`），套 Warm Ink 设计系统 + `05-内容创作标准.md`。
2. `App.tsx` 顶部加 `const MethodPage = lazy(() => import('../pages/MethodPage'))`，并在 `<Route path="*">` **之前**插入 `<Route path="/method" element={<MethodPage />} />`。
3. 导航入口：
   - `/method`：首页 CTA 已就位，无需额外；可选在 Sidebar/TopBar 加常驻入口。
   - `/problems`、`/about`：**当前无任何入口**，需在 Sidebar / TopBar / 页脚补链接。
4. 若某页尚未做好，保持不加路由（继续 404）——**不要用占位组件回填**。
