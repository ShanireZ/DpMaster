# 01 · 项目方案（DP 教学网站）

> **一句话**：做一个正常风格（非修真）、几何动态渐变视觉、精讲 + 重交互的**动态规划教学网站**，覆盖 A–G 七大 DP 家族，每类型 2–3 例题 + 3 练习，**全部洛谷原生题**；React + Vite 构建，产出可部署静态文件。主看设备 = 笔记本电脑。

- 目录：`D:\WorkSpace\dp`（与 cpplearn / cpplearn-animation-lab **无关**的独立项目）
- 配套文档：`02-内容架构与洛谷选题.md`（题库） · `03-设计与交互规范.md`（视觉/交互） · `04-技术栈与素材清单.md`（技术/许可证/素材）

---

## 1. 定位与目标

- **受众**：有 C++ 基础、系统学 DP 的算法竞赛学习者（NOIP/CSP/省选梯度）。
- **核心价值**：用**可操作、可改值的演示动画 + 互动小游戏**把 DP 的"状态、转移、无后效性、优化"讲出直觉，再用**成体系的洛谷原生例题/练习**落地。
- **风格**：正常教学风（去修真叙事）；几何动态渐变外壳华丽，正文层高对比强可读（双层分离，见 `03`）。
- **形态**：静态网站，`vite build` 产出 `dist/` 直接部署。

---

## 2. 范围（做什么 / 不做什么）

**做**
- 7 部分 × 37 类型，每类型：精讲 + ≥1 可改值演示 + 2–3 例题 + 3 练习；每部分 1 个互动小游戏。
- 全部例题/练习为洛谷原生 P/B 题（见 `02`，已逐题核验）。
- DP 通用方法论页 + 题目索引页（可筛选）+ 关于/资源许可页。
- 响应式（笔记本优先），`prefers-reduced-motion` 降级，可访问性基线。

**不做（本期）**
- **不做浏览器内 C++ 在线运行/判题**（默认：代码高亮 + 复制 + "在洛谷打开 Pxxxx" 链接）。如需在线运行是另一个量级的工程，单独立项。
- 不做账号/后端/进度云同步（进度只用 localStorage）。
- 不搬运洛谷题面全文（只做题面摘要 + 链接，尊重版权）。
- 不引入任何付费/禁再分发资源（见 `04`）。

## 4. 视觉与交互总览（详见 `03`）

- **几何动态渐变风**，双层分离：舞台层（首页/封面/背景）华丽，内容层（正文/代码/公式）克制高可读。
- **七部分渐变身份**：每部分一套专属渐变 + 几何母题（母题映射该 DP 结构），形成路标。
- **DP 可视化引擎**（全站复用）：渲染数组/表格/树/比特点阵，逐格填充、当前/依赖/被选高亮、KaTeX 方程联动、**可改值实时重算重播**、播放/单步/scrub、2D↔1D 与顺推/逆推对照。
- **每部分 1 个互动小游戏**（LIS 接龙 / 装包大师 / 合并石子 / 幂次加速器 / 换根巡礼 / 舞会邀请 / 棋盘布阵·TSP），可"一键展示 DP 最优解"对照玩家直觉。
- 反"卡片堆叠"：编辑式非对称 + 分栏（讲解↔演示）+ 滚动钉住叙事 + 有目的 bento。

## 5. 技术架构总览（详见 `04`）

- **React + Vite**，React Router 路由分包懒加载；产出静态 `dist/`。
- **动画**：GSAP（全插件免费）主力 + Motion（组件级）；**WebGL 仅 hero 一处**（r3f 只留给 3D 树演示）。
- **公式** KaTeX（构建期预渲染）· **代码高亮** Shiki（构建期预高亮，用户侧 0 JS）。
- **字体** 得意黑（标题）+ 思源黑体/Noto Sans SC（正文，**子集化**）+ Space Grotesk（拉丁）· **图标** Lucide · **音效** Kenney CC0。
- 全部依赖许可证已核实（MIT / OFL / CC0 / GSAP 免费），**教育站合规**。

## 6. 信息架构与页面结构

**站点地图**
```
/                首页：hero 几何渐变 + “DP 全景图”交互导航 + 项目简介
/method          DP 通用方法论：状态设计四步 · 无后效性 · 找转移 · 复杂度 · 优化路线图
/a … /g          七个部分页
   /b            背包 DP 部分封面（专属渐变+几何）+ 类型列表
   /b/01         01 背包（类型页：精讲→演示→例题→练习）
   /b/02 …       完全/多重/分组/混合/二维费用/依赖/综合/辨析
/problems        题目索引（可筛选：部分/类型/难度/来源年份/例题·练习）
/about           关于 · 如何使用 · 资源许可与致谢
```

**部分页（`/x`）解剖**：封面舞台（渐变+几何母题，预告该 DP 结构）→ 本家族"是什么/何时用/为何有效"→ 类型卡片列表 → 部分互动游戏入口 → 小结。

**类型页（`/x/nn`）解剖**（精讲 + 重交互统一模板）：
1. 精讲：要解决什么 → 状态定义 → 转移方程(KaTeX) → 复杂度 → ⚠️ 常见陷阱 / 💡 本质。
2. 可改值演示（分栏或滚动钉住）。
3. 例题 2–3：题面摘要 → 状态定义 → 转移方程 → 复杂度 → C++代码(ShanireZ 风, Shiki 高亮, 复制/洛谷链接) → 陷阱。
4. 练习 3：题号(渐变徽标) · 考点 · 提示 · 洛谷链接。
5. 上一类型 / 下一类型 导航。

**全局**：左侧部分导航（A–G 各带渐变）· 顶部进度/面包屑 · `⌘K` 搜题跳转 · 深色/浅色阅读切换 · localStorage 进度。

## 7. 构建路线图（并行 agent 策略）

> 约束：**不新增任何 branch / worktree**。多 agent 在**同一工作树**上按**互斥文件归属**并行（每部分独占自己的 `routes/<part>` `content/<part>` `games/<part>`），共享骨架先行、由主 agent 统一集成，规避冲突。

**Phase 0 · 脚手架（串行，主 agent）**
- Vite + React + Router + 设计令牌（`tokens/`：配色/字体/间距/7 渐变身份）。
- 共享组件：布局原语、代码面板(Shiki)、公式(KaTeX)、题目卡、信息框、几何渐变背景组件。
- **DP 可视化引擎骨架**（步骤生成器接口 + 数组/表格/树/比特点阵视图 + 播放控制）。
- 字体子集化流水线、`prefers-reduced-motion` 降级基建、性能基线。

**Phase 1 · 样板部分（串行，主 agent）**
- 选 **B 背包 DP**（类型最丰富、演示最有代表性：2D↔1D、顺推/逆推 bug 对照）端到端做完 1 个类型作**范式模板**（精讲+演示+例题+练习 全流程 + 装包游戏雏形），锁定组件契约与内容模板。

**Phase 2 · 六部分并行（多 agent，互斥文件）**
- 每 agent 领 1 个部分（A / C / D / E / F / G），严格套用 Phase 1 的模板与设计系统，产出该部分全部类型内容 + 各类型演示 + 部分游戏。
- B 部分补齐其余类型。E 换根与 F 树形有内容关联，安排相邻 agent 或同一 agent 处理边界（见 `02` §9 复用索引）。
- 主 agent 持续集成、统一风格、消解跨部分复用重复。

**Phase 3 · 全局与打磨（串行，主 agent）**
- 首页 hero + "DP 全景图"交互导航 + 方法论页 + 题目索引 + 关于页。
- 性能通关（代码分割/离屏暂停/字体子集/构建期高亮预渲染）、笔记本视口走查（1366/1440/1536）、可访问性、`prefers-reduced-motion`。
- **洛谷链接与原生性终检**（逐题确认 P/B 前缀可达）、`vite build` 产出 `dist/` 部署预演（用 preview 工具跑 dev server 截图取证）。

## 10. 开放问题（建设前请拍板）

1. **范围**：确认做**全量**（7 部分 37 类型 ~110–130 题），还是**先做精简版**（如 E 换根并入 F 作专题、或每类型练习先做 2 道）？
2. **建设方式**：先做 **1 个垂直切片**（脚手架 + B 部分 1–2 类型）给你验收视觉/交互/内容手感，通过后再并行全量；还是直接全量推进？（**推荐垂直切片先行**，降返工风险）
3. **正文基调**：默认**深色沉浸底 + 浅色阅读模式可切**；若你更想要**浅色为主**（长文阅读更稳），我调整设计令牌。
4. **站点命名/品牌**：是否要一个站名（如"DP 图谱 / DP Lab / 动规演武场(去修真则改中性名)"）与 favicon 基调？没有的话我起一个中性名。
5. **诚实缺口确认**：LCIS 以概念推导承载、打家劫舍以等价原生题承载——认可否？
